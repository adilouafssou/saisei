## 2. LangGraph アーキテクチャ

### ステートスキーマ（Pydantic V2 — 概念）

```
SaiseiState:
  # identity
  tdb_code: str            # 7 digits
  hojin_bango: str         # 13 digits
  company_profile: CompanyProfile | None

  # financials
  shisanhyo: list[TrialBalance]      # monthly, J-GAAP, JPY int
  working_capital_gap: int | None    # JPY int (Shikin Kuri)

  # macro
  boj_rate_curve: list[RatePoint]
  settlement_metrics: SettlementMetrics | None

  # assessment
  ews_score: float | None
  fsa_classification: FsaClass | None   # Joyo | Yoi Kanri | Yukyo Guchi

  # turnaround
  proposed_strategies: list[Strategy]
  approved_strategy: Strategy | None    # set after HITL
  keikakusho_draft: str | None

  # control
  errors: list[str]
  messages: list[AnyMessage]
```

### フローチャート（ノード + 条件付きエッジ）

```
        START
          │
          ▼
     [intake]  ── resolves TDB code + Hojin Bango, loads profile
          │
          ▼
   [core_banking fetch]  ── monthly Shisanhyo (via mocks)
          │
          ▼
     [ews_agent]  ── compute EWS score from trend signals
          │
          ▼
     [macro_agent] ── BOJ rates → settlement (T+1/T+2) → Shikin Kuri gap
          │
          ▼
    [classifier]  ── FSA class: Joyo / Yoi Kanri / Yukyo Guchi
          │
   ┌──────┴───────────────── conditional edge (on fsa_classification) ─────────┐
   │                                                                            │
   ▼ Joyo (Normal)                                          Yoi Kanri / Yukyo Guchi ▼
  [END: monitor only]                                              [strategist]
                                                                        │
                                                                        ▼
                                                            ╔═══════════════════════╗
                                                            ║ [hitl_negotiation]    ║  ← interrupt()
                                                            ║  human reviews &      ║
                                                            ║  negotiates strategy  ║
                                                            ╚═══════════════════════╝
                                                                        │
                                              conditional edge (on resume Command)
                                       ┌────────────────┬───────────────────────────┐
                                       ▼ approve        ▼ revise                     ▼ reject
                               [plan_writer]      back to [strategist]          [END: escalate]
                                       │
                                       ▼
                            generate Keikakusho draft
                                       │
                                       ▼
                                     END
```

- **interrupt ノード：** `hitl_negotiation` はグラフを一時停止し、提案された戦略をバンカーに提示します。
  実行は `Command(resume={...})` でバンカーの判断を受け取った後にのみ再開されます。
- **チェックポインター：** Postgres（`psycopg`）が interrupt をまたいでステートを永続化します。

## 3. データフロー

```
  Reflex UI ──HTTP──▶ Backend API (FastAPI hosting LangGraph)
                          │
                          ▼
                   LangGraph StateGraph
                          │  (nodes call data layer)
                          ▼
                   MockDataProvider  ◀── deterministic JSON fixtures
                   (core_banking / tdb / edinet_macro)
                          │
          state persisted ▼  Postgres checkpointer
                          │
   interrupt() ──▶ pause ──▶ Reflex negotiation_panel renders strategies
                          ◀── banker decision (approve/revise/reject)
          Command(resume) ▼
                   plan_writer ──▶ Keikakusho draft ──▶ Reflex dashboard
```

後から、モックレイヤーを実際のコアバンキング・TDB・EDINET クライアントに差し替えられます。
同じインターフェースの背後に差し込むだけで、グラフ変更は不要です。

## 4. 完了の定義

- [ ] `make setup` が `uv` をインストールし、コンテナをビルドし、Postgres にシードデータを投入する。
- [ ] `make run` が docker-compose 経由で web + api + postgres + redis を起動する。
- [ ] `make verify` が通過する：`ruff` クリーン、`mypy --strict` クリーン、`pytest` グリーン。
- [ ] `.gitlab-ci.yml` がすべての MR で `make verify` を実行する。
- [ ] MockDataProvider が決定論的な J-GAAP ペイロードを返す。愛知メーカーフィクスチャ（原価高騰 + 価格転嫁失敗 → 資金繰り赤字）を含む。
- [ ] LangGraph がエンドツーエンドで実行される：intake → EWS → macro → classify → （HITL）→ 経営改善計画書ドラフト。
- [ ] HITL interrupt が `Command(resume=...)` と Postgres チェックポインティングで正しく一時停止・再開する。
- [ ] FSA 区分が正常・要注意・要管理に厳格に限定されている。
- [ ] 円が `int` として `¥150,000,000` スタイルのフォーマットで全体を通じてレンダリングされる。
- [ ] Reflex UI が EWS ダッシュボード、試算表テーブル、交渉パネルを表示する。
- [ ] リポジトリにシークレットがない。`.env.example` が必要なすべての環境変数を文書化している。
